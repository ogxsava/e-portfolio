How I built this website

Stack & workflow

I wanted a lightweight blog without spending weeks on infrastructure. Here is what runs this site today.

Summary

  • Type: Static HTML and CSS (no CMS)
  • Hosting: GitHub Pages
  • Repository: github.com/ogxsava/e-portfolio
  • New post: ~15–30 minutes (new HTML file, images, one line on the home page)

Why static files

Plain files are easy to version in Git, fast to load, and painless to back up. I can open any post in an editor, drop photos in images/, and publish with a git push. No database, no plugin updates.

Structure

├── index.html       # Home — list of posts
├── about.html
├── css/style.css
├── posts/           # One file per article
└── images/

Publishing

  1. Add a new posts/my-article.html.
  2. Link it from index.html.
  3. Push to GitHub — Pages rebuilds automatically.
